# 数据源表格管理 PRD

## 1. 背景

Canal Plus 当前数据源管理需要从连接配置详情页，调整为更接近成熟后台管理系统的数据源列表页。

本次改版参考用户提供的截图效果：顶部筛选、表格列表、行内操作、右上角新增和刷新、底部分页。

本功能仍然只覆盖数据源连接配置、连接测试和基础管理，不提供数据库客户端能力。

## 2. 目标

### 2.1 用户目标

- 管理员可以快速查看全部数据源。
- 管理员可以从表格中直接新增、测试、编辑和删除数据源。
- 运维人员可以快速识别数据源类型、Host、版本、创建时间和连接状态。
- 只读用户可以查看数据源列表，但不能执行测试或变更操作。

### 2.2 产品目标

- 将数据源管理页调整为高信息密度表格形态。
- 保留已有连接测试和保存前校验规则。
- 避免引入 SQL 查询、表结构浏览、数据预览等数据库客户端能力。

## 3. 对标产品和参考点

### 3.1 对标来源

用户提供的截图中的数据源列表管理页。

### 3.2 参考点

- 顶部筛选区包含类型筛选、全部筛选和查询按钮。
- 表格列包含数据源 ID、类型、Host、版本号、创建时间、操作。
- 数据源 ID 单元格使用主标题和副描述两行展示。
- 类型列使用数据库类型图标或短标签。
- Host 列展示内网标识和 `host:port`。
- 操作列展示 `测试连接` 和 `更多`。
- 页面右上角展示 `新增数据源` 和刷新按钮。
- 底部展示总数、页码、每页条数和跳页。

### 3.3 差异点

- 首期只实现 Canal Plus 已支持的数据源能力。
- 如果后端首期只支持 MySQL，类型筛选只展示 `MySQL` 和 `全部`。
- 如果接口暂不返回版本号，版本号列显示 `-`。
- 不实现截图中 Oracle、SQL Server、PostgreSQL、Hologres、TiDB、OceanBase、SAP HANA、KingbaseES、SSH 等真实连接能力，除非后端已支持。

## 4. 用户角色

### 4.1 平台管理员

- 可以查看数据源表格。
- 可以新增数据源。
- 可以编辑数据源。
- 可以删除数据源。
- 可以测试连接。
- 可以使用筛选、查询、刷新和分页。

### 4.2 运维人员

- 可以查看数据源表格。
- 可以测试连接。
- 可以使用筛选、查询、刷新和分页。
- 不可以新增、编辑、删除，除非权限系统另有配置。

### 4.3 只读用户

- 可以查看数据源表格。
- 可以使用筛选、查询、刷新和分页。
- 不可以新增、编辑、删除。
- 不可以测试连接。

## 6. 页面入口

### 6.1 入口位置

- 左侧导航：`数据源`。

### 6.2 默认页面

- 进入后展示数据源表格。
- 默认筛选条件为：
  - 类型：`全部`。
  - 状态：`全部`。
  - 页码：第 1 页。
  - 每页条数：`20 条/页`。

### 6.3 主要操作入口

- `查询`：按当前筛选条件加载数据。
- `新增数据源`：打开新增数据源面板。
- `刷新`：重新加载当前筛选条件下的数据。
- `测试连接`：测试当前行数据源连接。
- `更多`：展开行操作菜单。

## 7. 页面结构

### 7.1 页面布局

页面采用后台管理表格布局。

- 顶部工具区。
- 表格区。
- 底部分页区。
- 新增/编辑面板。

### 7.2 顶部工具区

左侧包含：

- 类型下拉。
- 状态下拉。
- `查询` 按钮。

右侧包含：

- `新增数据源` 按钮。
- 刷新图标按钮。

要求：

- 工具区高度固定。
- 控件横向排列。
- 小屏可换行，但表格区允许横向滚动。
- 文案保持简短。

### 7.3 表格列

#### 7.3.1 数据源 ID

展示：

- 主标题：数据源名称或数据源 ID。
- 副标题：描述、Host 摘要或 `No description`。
- 信息图标：用于后续扩展详情提示。

展示规则：

- 主标题单行展示，超长省略。
- 副标题单行展示，超长省略。
- 不展示密码。
- 不展示完整 DSN。

#### 7.3.2 类型

展示：

- 数据库类型图标或短标签。

首期规则：

- MySQL 展示 MySQL 图标或 `MySQL`。
- 未知类型展示 `-`。

#### 7.3.3 Host

展示：

- 内网标识：`内`。
- Host 和端口：`host:port`。

规则：

- Host 超长允许换行或省略。
- 不展示用户名密码拼接串。

#### 7.3.4 版本号

展示：

- 数据库版本号。

规则：

- 有版本号时展示原始版本文本。
- 版本文本较长时允许换行。
- 无版本号时展示 `-`。

#### 7.3.5 创建时间

展示：

- 创建时间。

格式：

```text
YYYY-MM-DD HH:mm:ss
```

#### 7.3.6 操作

展示：

- `测试连接`。
- `更多`。
- 向下箭头。

交互：

- 点击 `测试连接` 后测试当前行。
- 点击 `更多` 后展开菜单。
- 菜单包含 `编辑`、`删除`。
- 删除前必须二次确认。

## 8. 核心流程

### 8.1 查询数据源

1. 用户进入 `数据源` 页面。
2. 系统加载默认筛选条件下的数据源列表。
3. 用户修改类型或状态筛选。
4. 用户点击 `查询`。
5. 系统加载符合条件的数据。
6. 表格刷新。
7. 分页回到第 1 页。

### 8.2 刷新数据源

1. 用户点击刷新按钮。
2. 系统按当前筛选条件和页码重新加载数据。
3. 刷新期间按钮进入 loading 或 disabled 状态。
4. 刷新成功后更新表格。
5. 刷新失败后显示 `加载失败`。

### 8.3 新增数据源

1. 管理员点击 `新增数据源`。
2. 系统打开新增数据源面板。
3. 用户填写连接参数。
4. 用户点击 `测试`。
5. 测试成功后允许保存。
6. 用户点击 `保存`。
7. 系统保存数据源。
8. 保存成功后关闭面板。
9. 系统刷新列表并选中新数据源所在页。

### 8.4 测试连接

1. 用户点击某行 `测试连接`。
2. 当前行测试按钮进入 loading 状态。
3. 系统调用已保存数据源测试接口。
4. 测试成功后提示 `测试通过`。
5. 测试失败后提示 `测试失败`。
6. 系统更新连接状态、最近测试时间和测试结果。

### 8.5 更多菜单

1. 用户点击某行 `更多`。
2. 系统展开菜单。
3. 管理员可见：
   - `编辑`。
   - `删除`。
4. 运维人员和只读用户按权限隐藏或禁用无权限项。
5. 用户点击菜单外部或按 Esc 时关闭菜单。

### 8.6 编辑数据源

1. 管理员点击 `更多` 中的 `编辑`。
2. 系统打开编辑面板。
3. 系统展示已有配置，但不展示密码明文。
4. 如果只修改名称、用途、备注，可以直接保存。
5. 如果修改类型、主机、端口、用户名、密码、默认库，测试状态变为 `过期`。
6. 连接字段变更后必须重新测试成功才能保存。

### 8.7 删除数据源

1. 管理员点击 `更多` 中的 `删除`。
2. 系统展示确认提示。
3. 用户确认后删除。
4. 删除成功后提示 `已删除`。
5. 系统刷新当前页。
6. 如果当前页为空，自动回到上一页或展示空状态。

## 9. 筛选和分页规则

### 9.1 类型筛选

- 默认：`全部`。
- 首期选项：
  - `全部`。
  - `MySQL`。

### 9.2 状态筛选

- 默认：`全部`。
- 选项：
  - `全部`。
  - `未测`。
  - `可用`。
  - `失败`。
  - `过期`。

### 9.3 查询按钮

- 点击后应用筛选。
- 查询期间 disabled。
- 查询成功后刷新表格。
- 查询失败显示 `加载失败`。

### 9.4 分页

底部展示：

- 总数：`共 N 条`。
- 上一页。
- 页码。
- 下一页。
- 每页条数选择。
- 跳页输入框。
- `页` 文案。

默认：

- 每页 `20 条/页`。
- 当前页为 `1`。

规则：

- 页码超出范围时自动修正到最后一页。
- 筛选条件变化并点击查询后回到第 1 页。
- 每页条数变化后回到第 1 页。

## 10. 状态设计

### 10.1 连接状态

- `未测`：未执行过连接测试。
- `可用`：最近一次测试成功。
- `失败`：最近一次测试失败。
- `过期`：连接字段已修改但未重新测试。

### 10.2 页面状态

#### 空状态

- 文案：`暂无数据源`。
- 管理员展示 `新增数据源`。

#### 加载状态

- 表格展示 loading 或骨架。
- 查询、刷新、测试按钮禁用。

#### 查询成功

- 表格更新。
- 不额外弹成功提示。

#### 查询失败

- 文案：`加载失败`。
- 操作：`重试`。

#### 测试成功

- 提示：`测试通过`。
- 当前行状态更新为 `可用`。

#### 测试失败

- 提示：`测试失败`。
- 当前行状态更新为 `失败`。
- 保留原表格数据。

#### 保存成功

- 提示：`已保存`。

#### 删除成功

- 提示：`已删除`。

## 11. 权限规则

- 管理员可以新增、编辑、删除、测试。
- 运维人员可以查看和测试。
- 只读用户只能查看。
- 无权限操作隐藏优先。
- 如果保留禁用按钮，必须提供明确原因提示：`权限不足`。

## 12. 数据行为

### 12.1 输入数据

- 类型筛选。
- 状态筛选。
- 当前页码。
- 每页条数。
- 新增/编辑表单数据。

### 12.2 输出数据

- 数据源 ID。
- 数据源名称。
- 描述或备注。
- 类型。
- Host。
- 端口。
- 版本号。
- 创建时间。
- 连接状态。
- 最近测试时间。
- 最近测试结果。

### 12.3 持久化

- 新增、编辑、删除按现有数据源持久化规则执行。
- 密码必须密文保存。
- API 不返回密码明文。
- 编辑时密码为空表示不修改密码。

### 12.4 副作用

以下操作需要写操作日志：

- 新增数据源。
- 编辑数据源。
- 删除数据源。
- 测试连接。

测试连接不得：

- 创建表。
- 修改表。
- 查询业务表数据。
- 写入业务库。
- 修改业务库。

## 13. API 要求

### 13.1 获取数据源列表

```http
GET /api/datasources
```

说明：

- 如后端暂不支持服务端分页，前端可先基于完整列表分页。
- 如后端支持分页，应兼容类型、状态、页码、每页条数参数。

### 13.2 测试已保存数据源

```http
POST /api/datasources/{id}/test
```

### 13.3 新增数据源

```http
POST /api/datasources
```

### 13.4 编辑数据源

```http
PUT /api/datasources/{id}
```

### 13.5 删除数据源

```http
DELETE /api/datasources/{id}
```

## 17. 假设

- 首期仍以 MySQL 为主要支持类型。
- 版本号字段如果接口暂未返回，前端显示 `-`。
- 如接口暂不支持服务端分页，先使用前端分页。
- 截图中的多数据库图标作为视觉方向，不代表本次必须实现多数据库连接。
