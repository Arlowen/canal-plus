# 数据源管理 PRD

## 1. 背景

Canal Plus 当前需要重做数据源管理功能，使其更接近成熟数据库管理工具的连接配置体验，但产品定位不是通用数据库管理器。

本功能只覆盖数据源连接配置、连接测试和基础管理，不提供表结构浏览、SQL 查询、表修改或数据修改能力。

## 2. 目标

### 2.1 用户目标

- 管理员可以快速添加数据库数据源。
- 管理员可以在保存前测试连接是否可用。
- 管理员可以查看数据源连接状态和最近测试结果。
- 运维人员可以在控制台中识别哪些数据源可用、失败或未测试。

### 2.2 产品目标

- 将数据源管理改造成稳定、清晰、低风险的连接配置中心。
- 降低误操作风险，避免在 Canal Plus 中引入通用数据库客户端能力。
- 为后续节点、告警和运行状态能力提供可信的数据源基础信息。

## 3. 对标产品

### 3.1 对标范围

参考以下成熟数据库管理工具的连接配置体验：

- DBeaver。
- Navicat。
- DataGrip。

### 3.2 参考点

- 新建数据库连接时使用结构化表单。
- 连接参数填写后可先测试连接。
- 测试结果需要给出明确成功或失败反馈。
- 保存成功后连接出现在连接列表中。
- 编辑连接时不回显密码明文。

### 3.3 差异点

Canal Plus 不复刻成熟数据库管理工具的完整数据库操作能力，只参考连接管理相关交互。

明确不做：

- 表选择。
- 表结构查看。
- 表结构修改。
- SQL 编辑器。
- SQL 查询。
- 数据预览。
- 数据新增。
- 数据修改。
- 数据删除。
- 数据导入导出。

## 4. 用户角色

### 4.1 平台管理员

- 可以新增数据源。
- 可以编辑数据源。
- 可以删除数据源。
- 可以测试连接。
- 可以查看全部数据源状态。

### 4.2 运维人员

- 可以查看数据源列表和详情。
- 可以测试连接。
- 是否允许新增、编辑、删除由权限系统决定。

### 4.3 只读用户

- 可以查看数据源列表和详情。
- 不可以新增、编辑、删除。
- 不可以修改连接配置。

### 6.2 默认页面

## 8. 核心流程

### 8.1 新增数据源

1. 用户进入 `数据源` 页面。
2. 用户点击 `新增`。
3. 系统打开新增数据源面板。
4. 用户填写连接参数。
5. 用户点击 `测试`。
6. 系统校验必填字段。
7. 校验通过后，系统调用连接测试接口。
8. 测试期间 `测试` 按钮进入 loading 状态。
9. 测试成功后，系统展示 `可用`、响应耗时和测试时间。
10. 用户点击 `保存`。
11. 系统保存数据源。
12. 保存成功后关闭面板。
13. 系统刷新列表，并选中新建数据源。

### 8.2 编辑数据源

1. 用户在列表中选择数据源。
2. 用户点击 `编辑`。
3. 系统打开编辑面板。
4. 系统展示已有配置，但不展示密码明文。
5. 用户修改字段。
6. 如果用户修改连接字段，测试状态变为 `过期`。
7. 用户点击 `测试`。
8. 测试成功后，用户点击 `保存`。
9. 系统保存修改并刷新详情。

连接字段包括：

- 类型。
- 主机。
- 端口。
- 用户名。
- 密码。
- 默认库。

非连接字段包括：

- 名称。
- 用途。
- 备注。

如果只修改非连接字段，可以不重新测试直接保存。

### 8.3 删除数据源

1. 用户点击 `删除`。
2. 系统展示确认提示。
3. 用户确认删除。
4. 系统删除数据源。
5. 删除成功后刷新列表。
6. 如果删除的是当前选中项，系统自动选中下一条。
7. 如果列表为空，展示空状态。

### 8.4 测试已保存数据源

1. 用户选择一个已保存数据源。
2. 用户点击 `测试`。
3. 系统调用已保存数据源测试接口。
4. 测试期间按钮 loading。
5. 测试成功后更新状态为 `可用`。
6. 测试失败后更新状态为 `失败`。
7. 系统记录最近测试时间和测试结果。

## 9. 表单字段

### 9.1 名称

- 必填。
- 1 到 50 个字符。
- 建议同名时提示用户。

### 9.2 类型

- 必填。
- 首期默认支持 `MySQL`。
- 如暂不支持其他类型，不展示其他类型。

### 9.3 用途

- 可选。
- 选项：
  - `源库`。
  - `目标库`。
  - `通用`。

### 9.4 主机

- 必填。
- 支持域名。
- 支持 IPv4。
- 支持 IPv6。

### 9.5 端口

- 必填。
- 默认值：`3306`。
- 取值范围：1 到 65535。

### 9.6 用户名

- 必填。

### 9.7 密码

- 新增时必填。
- 编辑时不回显明文。
- 编辑时为空表示不修改密码。
- 需要提供修改密码入口或密码输入框。

### 9.8 默认库

- 可选。
- 仅作为连接默认 database/schema。
- 不触发表枚举。

### 9.9 备注

- 可选。
- 最多 200 个字符。

## 10. 测试连接规则

### 10.1 测试入口

- 新增面板中可以测试未保存的连接参数。
- 编辑面板中可以测试当前表单连接参数。
- 详情区可以测试已保存的数据源。

### 10.2 测试前校验

以下字段未填写时，不能发起测试：

- 名称。
- 类型。
- 主机。
- 端口。
- 用户名。
- 密码，新建场景必填。

### 10.3 测试中

- `测试` 按钮显示 loading。
- 禁止重复点击。
- 不阻塞用户关闭面板，但关闭时需要确认未保存变更。

### 10.4 测试成功

展示：

- 状态：`可用`。
- 响应耗时。
- 测试时间。

### 10.5 测试失败

展示：

- 状态：`失败`。
- 失败原因。
- `重试` 入口。

要求：

- 不清空表单。
- 错误信息必须脱敏。
- 不展示密码、完整 DSN 或敏感连接串。

### 10.6 测试结果过期

用户修改以下任一字段后，已有测试结果变为 `过期`：

- 类型。
- 主机。
- 端口。
- 用户名。
- 密码。
- 默认库。

### 10.7 保存前规则

- 新增数据源必须测试成功后才能保存。
- 编辑数据源时，如果修改连接字段，必须重新测试成功后才能保存。
- 编辑数据源时，如果只修改名称、用途、备注，可以直接保存。

## 11. 状态设计

### 11.1 连接状态

- `未测`：未执行过连接测试。
- `可用`：最近一次测试成功。
- `失败`：最近一次测试失败。
- `过期`：连接字段已修改，但未重新测试。

### 11.2 页面状态

#### 空状态

- 文案：`暂无数据源`。
- 主操作：`新增`。

#### 加载状态

- 列表展示 loading 或骨架屏。
- 操作按钮禁用。

#### 保存成功

- 提示：`已保存`。

#### 删除成功

- 提示：`已删除`。

#### 测试成功

- 提示：`测试通过`。

#### 测试失败

- 提示：`测试失败`。
- 展示失败原因。

#### 加载失败

- 文案：`加载失败`。
- 操作：`重试`。

## 12. 权限规则

- 管理员可以新增、编辑、删除、测试。
- 运维人员默认可以查看和测试。
- 只读用户只能查看。
- 无权限时隐藏或禁用对应操作。
- 禁用按钮需要有明确原因提示。

## 13. 数据行为

### 13.1 输入数据

- 名称。
- 类型。
- 用途。
- 主机。
- 端口。
- 用户名。
- 密码。
- 默认库。
- 备注。

### 13.2 输出数据

- 数据源 ID。
- 连接状态。
- 最近测试时间。
- 最近测试结果。
- 创建时间。
- 更新时间。

### 13.3 持久化

- 密码必须加密或密文保存。
- API 不返回密码明文。
- 编辑时密码为空表示不修改密码。

### 13.4 副作用

以下操作需要写操作日志：

- 新增数据源。
- 编辑数据源。
- 删除数据源。
- 测试连接。

测试连接不得：

- 创建表。
- 修改表。
- 查询业务表数据。
- 写入业务库。
- 修改业务库。

## 14. API 要求

### 14.1 获取数据源列表

```http
GET /api/datasources
```

### 14.2 新增数据源

```http
POST /api/datasources
```

### 14.3 编辑数据源

```http
PUT /api/datasources/{id}
```

### 14.4 删除数据源

```http
DELETE /api/datasources/{id}
```

### 14.5 测试未保存数据源

```http
POST /api/datasources/test
```

### 14.6 测试已保存数据源

```http
POST /api/datasources/{id}/test
```

### 14.7 测试成功响应

```json
{
  "success": true,
  "status": "available",
  "latencyMs": 42,
  "testedAt": "2026-05-28T10:30:00+08:00",
  "message": "Connection available"
}
```

### 14.8 测试失败响应

```json
{
  "success": false,
  "status": "failed",
  "testedAt": "2026-05-28T10:30:00+08:00",
  "message": "Access denied"
}
```

## 16. 日志要求

所有应用日志必须使用英文。

每条日志必须使用以下格式：

```text
[time][level][thread]message
```

要求：

- `level` 只能是 `info`、`warn` 或 `error`。
- `thread` 使用线程名、任务名、进程名或稳定的执行上下文名称。
- 日志不得包含密码。
- 日志不得包含完整 DSN。

## 17. 验收标准

- [ ] 用户可以进入数据源页面并看到数据源列表。
- [ ] 用户可以新增 MySQL 数据源。
- [ ] 必填字段缺失时不能测试或保存。
- [ ] 用户可以在保存前测试连接。
- [ ] 测试连接不会保存表单内容。
- [ ] 测试成功后展示状态、耗时、时间。
- [ ] 测试失败后展示脱敏错误，并允许重试。
- [ ] 修改连接字段后，测试状态变为 `过期`。
- [ ] 修改连接字段后必须重新测试成功才能保存。
- [ ] 只修改名称、用途、备注时可以直接保存。
- [ ] 保存成功后列表刷新并选中新数据源。
- [ ] 编辑数据源时不回显密码明文。
- [ ] 删除数据源前有确认。
- [ ] 页面中没有表选择、SQL 编辑、数据预览、数据修改入口。
- [ ] 测试连接不创建表、不修改表、不查询业务表数据。
- [ ] 后端日志符合 `[time][level][thread]message` 格式。
- [ ] 前端文案保持极简。

- 默认库只是连接参数，不触发表枚举。
- 管理员可新增、编辑、删除。
- 运维人员默认可查看和测试。
- 只读用户只能查看。
- 对标产品只参考连接配置和测试连接流程，不参考数据库管理能力。
